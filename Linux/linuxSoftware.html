Linux Software Management

Software in Linux is organized in packages. These contain all of the files a given program needs to run, and a list of other packages it depends on to run.

These packages are managed by, you guessed it, a package manager. The one we deal with most often is APT, the Advanced Package Tool. It comes pre-installed on all Debian-based Linux distributions. This includes Ubuntu and Mint, the two we use in CyberTitans.

What is APT?

APT is the package manager we use on Ubuntu and Mint images. It is how we install, update, and uninstall software. It is run from the command line, and has a general syntax of:

apt action package

Depending on what action you supply, you might not need a specific package name.

(Un)installation

Installing and uninstalling packages with APT is simple. To install, it's sudo apt install package, where package is the name of the package.

Take note, however, that the technical package name is not always the same as the name displayed in the app list. Usually, it's similar, but you can look up software to find its technical name.

To uninstall, it's very similar. sudo apt remove package will remove the specified package. Replacing remove with purge will remove not just the package, but any packages it depends on that no longer have a use.

Generally, remove works perfectly fine, and is a lot less risky than purge. I recommend only using purge if you're absolutely certain what you're doing. It can make quite a mess if misused.

Updating / Upgrading

To update software (called upgrading in apt) you use two commands.

The first,

sudo apt update

just updates the list of available packages from your default sources. This makes sure your computer knows which versions of software are available so that upgrades are not missed.

Then, after that has completed, run

sudo apt upgrade

This does the actual installation of the upgrades. It may take a while, so it's a good option for doing during lunch.
