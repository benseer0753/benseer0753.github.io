Pluggable Authentication Modules (PAM)

Updating / Installing PAM

update: libpam-runtime, libpam-modules, libpam-pwquality

faillock

In /etc/security/faillock.conf, add or edit the following lines:

deny = 5
unlock-time = 900
root_unlock_time = 60

pam_pwquality

module aguments override etc/security/pwquality.conf overrides etc/security/pwquality.conf.d/ files

For pam_pwquality, there is a hierarchy to which settings take priority. Module arguments in the common-* files override settings in etc/security/pwquality.conf. Settings in files in etc/security/pwquality.conf.d/ are overridden by either. It is recommended by the CIS to use a *.conf file in etc/security/pwquality.conf.d/ for "clarity, convenience, and durability."

Include the following lines:

difok = 2
minlen = 14
minclass = 0
dcredit = 0
ucredit = 0
ocredit = 0
lcredit = 0
maxrepeat = 3
maxsequence = 3
enforce_for_root

Remove all lines containing "dictcheck" and "enforcing". This, counterintuitively, enables them because their default values are 1.

pam_pwhistory

The following should all be parameters on the pam_pwhistory line in common-password:

remember=24 enforce_for_root use_authtok
