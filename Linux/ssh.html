Secure Shell (SSH)

SSH is a secure, encrypted way to log in to remote computers. It is useful for people working from home or for working on servers that don't have monitors or other interface tools. SSH is available to use on both Windows and Linux, but is much more useful on Linux due to the console being much more powerful there. In CyberTitans, we will only be configuring it on Linux unless told otherwise.

How to Access SSH Configuration

The configuration files for SSH are stored in one of two locations. The first is the file /etc/ssh/sshd_config. The other is in any files in /etc/ssh/ssh_config.d/ that end in ".conf".

These files should have their permissions set to 0700. You can set this using the following script:

Ensure that the files ~/ssh/config for each user has the correct permissions by setting it to 0600 using chmod.

The Settings

Most of these settings will likely be configured already, but we still need to check each one to make sure. If you add any lines, make sure they go above any Include or Match statements. Essentially, put them at the top of the file. When the settings are read, they work from top to bottom.

Protocol 2
Ciphers -3des-cbc,aes128-cbc,aes192-cbc,aes256-cbc,chacha20-poly1305@openssh.com
ClientAliveInterval 15
ClientAliveCountMax 3
DisableForwarding yes
GSSAPIAuthentication no
HostbasedAuthentication no
IgnoreRhosts yes
KexAlgorithms -diffie-hellman-group1-sha1,diffie-hellman-group14-sha1,diffie-hellman-group-exchange-sha1
LoginGraceTime 60
loglevel VERBOSE
MACs -hmac-md5,hmac-md5-96,hmac-ripemd160,hmac-sha1-96,umac-64@openssh.com,hmac-md5-etm@openssh.com,hmac-md5-96-etm@openssh.com,hmac-ripemd160-etm@openssh.com,hmac-sha1-96-etm@openssh.com,umac-64-etm@openssh.com,umac-128-etm@openssh.com
MaxAuthTries 4
MaxSessions 10
MaxStartups 10:30:60
PermitEmptyPasswords no
PermitRootLogin no
PermitUserEnvironment no
UsePAM yes

After making any changes run the following:

systemctl restart sshd

This will restart the sshd service and apply any changes you have made.
