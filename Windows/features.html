Roles and Features

What are Roles and Features?

Roles define what type of server Windows computers are. This includes things like HTML webservers, file share servers, or domain controllers managing other computers on the network.

Features are generally a bit smaller than roles. They work as little bonus pieces of functionality for the computer. For example, BitLocker drive encryption is a feature. It works in addition to any other roles and features the computer has and doesn't completely change its function.

Sometimes there will be roles and features installed on a computer that do not need to be installed. These can present a security risk. The more unnecessary features on a computer, the more likely it is that at least one will have a security vulnerability.

Besides being a security risk, extra roles and features running in the background can slow down the computer. In a competition, this is annoying; in a real work environment, this could lead to loss of productivity or past work.

Adding / Removing Roles and Features

Adding and removing roles and features is slightly different between Server and workstation Windows. On regular windows, you want to open "appwiz.cpl" in the run dialogue or find it in control panel.

On Windows Server, they can be managed in the Server Manager window. Click on the "Manage" tab in the top-right, then on "Add/Remove Roles and Features". From there, follow the prompts on-screen.

Roles to disable

All of them. Unless explicitly stated in the README, it shouldn't be there. Things like Telnet or FTP are common to see, and should always be removed (unless specified) due to security flaws.
