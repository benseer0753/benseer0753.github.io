editing /etc/pam.d/common-password

minlen=10 <- this is generally enough, I think. It's what I usually use on windows. if you don't get points, check you did it right then set it higher.

retry=5 <- I think this is good, if it works like windows password limits. Too few means you get locked out for a typo, too many doesn't stop malactors in time

auth required pam_tally2.so deny=5 unlock_time=3600 even_deny_root <- this locks you out of the account if you mess up 5 times.

PASS_MAX_DAYS should be 90, I think? Set it to the same as whatever windows has as default.

PASS_MIN_DAYS should be 30, that's what they like it set to

PASS_MIN_LEN also like 10-ish? I think these caps ones are for new account creation.
